# My Portfolio — Setup Guide

This README explains how to run the portfolio locally and how to configure Supabase (database, storage, and policies).

Prerequisites
- Node.js (LTS) — https://nodejs.org/
- PowerShell (Windows)

Quick start (Windows PowerShell)
1. Open PowerShell in: `C:\my-portfolio`
2. Install dependencies:
   ```powershell
   npm install
   ```
3. Create a local .env from the example and edit it:
   ```powershell
   Copy-Item .env.example .env
   notepad .env
   ```
   Add your Supabase values:
   ```
   VITE_SUPABASE_URL=your-project-url
   VITE_SUPABASE_ANON_KEY=your-anon-key
   VITE_CONTACT_EMAIL=your-email@example.com
   ```

Run the app
```powershell
npm run dev
# Visit http://localhost:5173
```

Supabase setup (summary)
1. Create a project at https://supabase.com
2. In Project → Settings → API copy:
   - Project URL
   - Anon public key
3. In Supabase SQL Editor run the SQL below (or paste your adapted version) to create tables, enable RLS, and add example data.

Example SQL (tables, RLS & basic policies)
```sql
-- projects table
CREATE TABLE public.projects (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at timestamptz DEFAULT now() NOT NULL,
  "Title" text,
  "Description" text,
  "Img" text,
  "Link" text,
  "Github" text,
  "Features" jsonb,
  "TechStack" jsonb,
  "Images" text
);

-- certificates table
CREATE TABLE public.certificates (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at timestamptz DEFAULT now() NOT NULL,
  "Img" text
);

-- portfolio_comments table (UUID)
CREATE TABLE public.portfolio_comments (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  content TEXT NOT NULL,
  user_name VARCHAR(255) NOT NULL,
  profile_image TEXT,
  is_pinned BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Enable RLS
ALTER TABLE public.projects ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.certificates ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.portfolio_comments ENABLE ROW LEVEL SECURITY;

-- Public read policies
CREATE POLICY "Public read projects" ON public.projects FOR SELECT TO public USING (true);
CREATE POLICY "Public read certificates" ON public.certificates FOR SELECT TO public USING (true);
CREATE POLICY "Public read comments" ON public.portfolio_comments FOR SELECT TO public USING (true);

-- Allow public inserts to comments but prevent creating pinned comments
CREATE POLICY "Public insert comments" ON public.portfolio_comments FOR INSERT TO public WITH CHECK (is_pinned = false);
```

Storage (profile images / project images)
1. In Supabase → Storage create a public bucket (e.g. `portfolio` or `profile-images`).
2. Add policies to allow public reads and public uploads for that bucket:
```sql
CREATE POLICY "Public read storage" ON storage.objects FOR SELECT USING (bucket_id = 'portfolio');
CREATE POLICY "Public insert storage" ON storage.objects FOR INSERT WITH CHECK (bucket_id = 'portfolio');
```

Comments & moderation (optional)
- To require moderation, add an `approved` boolean to the comments table and change the SELECT policy to only show approved comments.
- Example:
```sql
ALTER TABLE public.portfolio_comments ADD COLUMN approved BOOLEAN DEFAULT FALSE;
DROP POLICY "Public read comments" ON public.portfolio_comments;
CREATE POLICY "Public read approved comments" ON public.portfolio_comments FOR SELECT TO public USING (approved = true);
```

Adding projects (via Supabase UI)
- Go to Supabase → Table Editor → public.projects → Insert row
- For JSON fields use valid JSON arrays, e.g. Features: ["Feature 1", "Feature 2"]
